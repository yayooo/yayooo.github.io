总结一下工作室刚开始做项目的几个步骤吧
基本操作和概念看廖雪峰git教程
我只讲一讲刚刚接触项目时会用到的和可能遇到的问题的解决方法

注册账号

*工作室git的账号和github的账号是不一样的
记住id和邮箱

安装git

我用的廖雪峰git教程里提到的国内镜像，全程默认安装。
然后打开git bash输入下面两行，这里的id和email就是之前创建的工作室git账号
$ git config –global user.name “你的id”
$ git config –global user.email “你的email”

添加ssh

输入
$ ssh-keygen -t rsa -C “你的email”
一路回车，会得到两个文件id_rsa和id_rsa.pub，路径在回车期间出现过，可以回过去看。
打开id_rsa.pub（最好不要用记事本）全文复制
网页登录git账号->页面右上角头像下拉栏->用户设置->左侧管理ssh密钥->增加密钥（蓝色）->粘贴->增加密钥（绿色）

添加远程库

网页左上角控制面板右侧参与协作的仓库右侧头像下方的派生
git remote add 自定义的区别于origin的名字，表示不同的来源 对应来源的地址（对应此项目就是http://git.starstudio.org/regiondavid/retire-mobile.git）
然后进行正常的拉取远程仓库操作，只是区别为要将 origin 换成 上边自定义的名字。eg：git fetch test branch:branch。

每次提交前一定要记得执行 git fetch / git pull 操作，先解决一下可能有的冲突！

git协作规范

每个人把这个仓库派生（fork）到自己的账号下，将自己的账号下的仓库拉到本地进行开发。

每个人新建一个分支（切记：分支名用功能去命名，多个词语时使用下划线连接），并在自己的分支下进行开发，并及时将自己的分支推送到自己账号下的远程仓库的对应分支。当某个功能开发完毕后，从自己的账号下的仓库发起合并请求，请求当前分支合并到原始仓库的 master 分支。

每次开发前记得先去拉去 master 分支最新的更新，并合并 master 分支到自己开发的分支，如有冲突，自行解决冲突
