总结一下工作室刚开始做项目的几个步骤吧
基本操作和概念看廖雪峰git教程
我只讲一讲我刚刚接触项目时会用到的和可能遇到的问题的解决方法
没有处理的问题等开学后问学长学姐弄清楚后再更新

注册账号

*工作室git的账号和github的账号是不一样的
记住id和邮箱

安装git

我用的廖雪峰git教程里提到的国内镜像，全程默认安装。
然后打开git bash输入下面两行，这里的id和email就是之前创建的工作室git账号

$ git config --global user.name "你的id"
$ git config --global user.email "你的email"

添加ssh

输入

$ ssh-keygen -t rsa -C "你的email"

一路回车，会得到两个文件id_rsa和id_rsa.pub，路径在回车期间出现过，可以回过去看。
打开id_rsa.pub（最好不要用记事本）全文复制
网页登录git账号->页面右上角头像下拉栏->用户设置->左侧管理ssh密钥->增加密钥->粘贴->增加密钥（绿色）

添加远程库

网页左上角控制面板->右侧参与协作的仓库->右侧头像下方的派生
*仓库左侧的仓库名处有“派生自xxx”才是自己派生的仓库
进入自己派生的仓库后复制右边的链接（最好是http的），然后输入下面两行，XXX为自定义的仓库名，http://XXX为之前复制的链接
*git中的复制粘贴不是ctrl+c，ctrl+v，我的是右键后copy，paste。

$ git remote add XXX http://XXX
$ git clone http://XXX

创建分支

*一个功能一个分支
创建并切换分支，分支名自定义，一般是功能的英文，多个单词用下划线连接。

$ git checkout -b xxx

下面两行前者是创建，后者是切换

$ git branch xxx
$ git checkout xxx

进行开发

$ git add xxxx

$ git commit -m "xxxx"

两个过程反复
开发完成后切换回master分支

$ git checkout master

*每次开发前要拉取master的最新更新并合并 master 分支到自己开发的分支，自行解决冲突后再开发。

提交并请求合并

每次提交前一定要记得执行 git fetch / git pull 操作，先解决一下可能有的冲突！然后再git push。
push成功后网页进入自己派生的仓库，点击左侧的绿色按钮发起合并请求。
